# C语言字符串与内存管理学习笔记

## 1. 哈希表实现中的内存管理问题

### 1.1 问题背景
在实现哈希表时，遇到字符串存储的内存管理问题：
```c
// 错误的浅拷贝方式
void put(ArrayHashMap *map, int key, char *value) {
    Pair *pair = malloc(sizeof(Pair));
    pair->key = key;
    pair->value = value;  // ❌ 只复制指针，存在野指针风险
}

// 正确的深拷贝方式
void put(ArrayHashMap *map, int key, char *value) {
    Pair *pair = malloc(sizeof(Pair));
    pair->key = key;
    pair->value = malloc(strlen(value) + 1);  // ✅ 分配新内存
    strcpy(pair->value, value);               // ✅ 复制字符串内容
}
```

### 1.2 深拷贝的必要性
**问题场景**：
1. **临时字符串修改**：原字符串被修改，哈希表数据被意外改变
2. **局部变量生命周期**：函数结束后局部字符串被销毁，产生野指针
3. **动态内存释放**：调用者释放内存，哈希表指向无效内存

**解决方案**：深拷贝确保哈希表拥有字符串的独立副本

## 2. C语言字符串基础概念

### 2.1 字符串的本质
```c
// 字符串就是以'\0'结尾的字符数组
char str[] = {'h', 'e', 'l', 'l', 'o', '\0'};
// 等价于
char str[] = "hello";
```

### 2.2 字符串的几种定义方式

#### 2.2.1 字符数组（栈上，可修改）
```c
char str1[] = "hello";
str1[0] = 'H';  // ✅ 可以修改
```

#### 2.2.2 字符指针指向字符串常量（只读）
```c
char *str2 = "hello";
// str2[0] = 'H';  // ❌ 错误！不能修改字符串常量
```

#### 2.2.3 动态分配（堆上，可修改）
```c
char *str3 = malloc(6);
strcpy(str3, "hello");
str3[0] = 'H';  // ✅ 可以修改
free(str3);     // ✅ 需要手动释放
```

### 2.3 字符串常量的内存布局
```c
char *str = "hello";  // "hello"是地址类型！
```
**内存分布**：
- `"hello"`存储在程序的**只读数据段**
- `"hello"`表达式的值是字符串首字符的地址
- `str`变量存储这个地址值

```
只读数据段：[h][e][l][l][o][\0] ← 地址0x401000
栈内存：   str = 0x401000
```

## 3. 字符串操作函数

### 3.1 必须掌握的字符串函数
```c
#include <string.h>

// 1. 长度计算（不包括'\0'）
int len = strlen("hello");  // len = 5

// 2. 字符串复制
char dest[10];
strcpy(dest, "hello");      // 复制整个字符串
strncpy(dest, "hello", 9);  // 最多复制9个字符

// 3. 字符串连接
char str[20] = "hello";
strcat(str, " world");      // str变成"hello world"

// 4. 字符串比较
if (strcmp(str1, str2) == 0) {  // 字符串相等
}

// 5. 字符串查找
char *pos = strchr(str, 'l');    // 找第一个'l'
char *pos = strstr(str, "llo");  // 找子串"llo"
```

### 3.2 内存分配相关
```c
// 为字符串分配内存的正确方式
char *allocateString(const char *source) {
    if (source == NULL) return NULL;
    
    int len = strlen(source);
    char *copy = malloc(len + 1);  // +1为'\0'留空间
    if (copy != NULL) {
        strcpy(copy, source);
    }
    return copy;
}
```

## 4. 数组名与指针的区别

### 4.1 核心区别
```c
char arr[10] = "hello";
char *ptr = "world";
```

| 特性 | 数组名 | 指针变量 |
|------|--------|----------|
| 本质 | 编译时常量 | 运行时变量 |
| 重新赋值 | ❌ 不可以 | ✅ 可以 |
| 算术运算 | ❌ 不可以 | ✅ 可以 |
| sizeof结果 | 整个数组大小 | 指针大小 |
| 内存位置 | 栈上数组 | 栈上指针变量 |

### 4.2 数组名的特性
- `arr`在大多数情况下等价于`&arr[0]`（首元素地址）
- `arr`本身不是变量，是编译时常量
- `arr`不能被重新赋值
- 函数参数中数组名会"退化"为指针

### 4.3 实例对比
```c
char arr[10] = "hello";
char *ptr = arr;

// 可以做的操作
ptr = "new";    // ✅ 指针可以重新赋值
// arr = ptr;   // ❌ 数组名不能重新赋值

ptr++;          // ✅ 指针可以移动
// arr++;       // ❌ 数组名不能移动

// sizeof的区别
sizeof(arr);    // 10（整个数组大小）
sizeof(ptr);    // 8（指针大小，64位系统）
```

## 5. const关键字的使用

### 5.1 const的含义
`const`是编译器约束，不是内存属性的描述：

```c
// 推荐写法：明确表示指向的内容不可修改
const char *str = "hello";
// 或者
char const *str = "hello";
```

### 5.2 各种const用法
```c
char arr[] = "hello";

const char *p1 = arr;        // p1指向的内容不能通过p1修改
char const *p2 = arr;        // 同上，写法不同
char *const p3 = arr;        // p3指针本身不能改变指向
const char *const p4 = arr;  // 既不能改变指向，也不能修改内容
```

### 5.3 const vs 只读内存
```c
// 字符数组：内存可写，const是编译器约束
char arr[] = "hello";
const char *constPtr = arr;
// constPtr[0] = 'H';  // ❌ 编译错误（const约束）
arr[0] = 'H';          // ✅ 直接修改原数组可以

// 字符串常量：内存本身只读
char *str = "hello";
str[0] = 'H';          // ❌ 运行时错误（只读内存）
```

## 6. 内存传参过程详解

### 6.1 传参的本质
内存就像小区，变量住在不同房子里，传参就是传"房子地址"：

**值传递**（传房子里的东西）：
```c
void func1(int x) {  // 把age房子里的25拷贝一份给x
    x = 30;          // 只改了x的值，原变量不变
}
```

**指针传递**（传房子地址）：
```c
void func2(int *x) { // 把变量房子的地址给x
    *x = 30;         // 通过地址找到房子，改里面的值
}
```

### 6.2 链表vs字符串传参对比

**链表操作**：我们要改变链表结构
```c
void addNode(Node **head, int value) {
    // head是指向"head指针房子"的地址
    // 我们要改的是head房子里存的地址（谁是头节点）
    Node *newNode = malloc(sizeof(Node));
    newNode->next = *head;
    *head = newNode;  // 改变head指针的值
}
```

**字符串操作**：我们要保存字符串内容
```c
void saveString(char **storage, char *input) {
    // 我们要为字符串内容找个新房子
    // 我们关心的是"指针指向的内容"（字符串是什么）
    *storage = malloc(strlen(input) + 1);
    strcpy(*storage, input);  // 复制内容到新房子
}
```

## 7. 栈内存数组初始化

### 7.1 初始化规则
```c
// 1. 声明但不初始化（垃圾值）
char dest[10];           // ❌ 内容是随机的！

// 2. 显式初始化为0
char dest[10] = {0};     // ✅ 所有元素都是0
char dest[10] = "";      // ✅ 第一个是'\0'，其余是0

// 3. 部分初始化
char dest[10] = "hi";    // ✅ 前3个是'h','i','\0'，其余是0
```

### 7.2 栈vs堆的初始化对比
```c
char stack[5];              // 栈上，未初始化，垃圾值
char stackInit[5] = {0};    // 栈上，初始化为0
char *heap = malloc(5);     // 堆上，未初始化，垃圾值
char *heapZero = calloc(5, 1); // 堆上，初始化为0
```

## 8. 内存管理最佳实践

### 8.1 内存管理原则
**谁分配，谁释放**：
```c
// 分配内存
char *str = malloc(strlen(source) + 1);
strcpy(str, source);

// 使用...

// 释放内存
free(str);
str = NULL;  // 避免悬挂指针
```

### 8.2 哈希表中的内存管理
```c
// 构造函数：初始化
ArrayHashMap* newArrayHashMap() {
    ArrayHashMap *map = malloc(sizeof(ArrayHashMap));
    for(int i = 0; i < MAX_SIZE; i++) {
        map->buckets[i] = NULL;  // 初始化为NULL
    }
    return map;
}

// 插入函数：深拷贝
void put(ArrayHashMap *map, int key, char *value) {
    Pair *pair = malloc(sizeof(Pair));
    pair->key = key;
    pair->value = malloc(strlen(value) + 1);  // 为字符串分配内存
    strcpy(pair->value, value);               // 深拷贝
    
    int index = hashFunction(key);
    if(map->buckets[index] != NULL) {
        free(map->buckets[index]->value);     // 释放旧字符串
        free(map->buckets[index]);            // 释放旧Pair
    }
    map->buckets[index] = pair;
}

// 析构函数：清理所有内存
void deleteArrayHashMap(ArrayHashMap *map) {
    for(int i = 0; i < MAX_SIZE; i++) {
        if(map->buckets[i] != NULL) {
            free(map->buckets[i]->value);     // 先释放字符串
            free(map->buckets[i]);            // 再释放Pair
        }
    }
    free(map);
}
```

## 9. 常见错误与解决方案

### 9.1 内存泄漏
```c
// ❌ 错误：忘记释放内存
char *str = malloc(100);
strcpy(str, "hello");
// 忘记free(str);

// ✅ 正确：及时释放
char *str = malloc(100);
strcpy(str, "hello");
// ... 使用str
free(str);
str = NULL;
```

### 9.2 悬挂指针
```c
// ❌ 错误：使用已释放的内存
char *str = malloc(10);
free(str);
strcpy(str, "hello");  // 未定义行为

// ✅ 正确：释放后置NULL
char *str = malloc(10);
free(str);
str = NULL;
if(str != NULL) {      // 检查后使用
    strcpy(str, "hello");
}
```

### 9.3 缓冲区溢出
```c
// ❌ 错误：没有检查缓冲区大小
char dest[5];
strcpy(dest, "hello world");  // 溢出

// ✅ 正确：使用安全函数
char dest[5];
strncpy(dest, "hello world", 4);
dest[4] = '\0';  // 确保以'\0'结尾
```

## 10. 总结要点

1. **字符串常量存储在只读内存区域，不能修改**
2. **字符数组分配在栈上，内容可以修改但地址固定**
3. **深拷贝是数据结构设计的重要原则，避免外部依赖**
4. **const是编译时约束，不改变内存的实际属性**
5. **数组名是地址常量，指针是地址变量**
6. **字符串操作必须考虑'\0'结束符的空间**
7. **内存管理遵循"谁分配谁释放"的原则**
8. **栈上变量不会自动初始化，需要显式初始化**

这些知识点是C语言字符串和内存管理的核心，掌握后能避免大部分内存相关的错误！
