# 环形链表检测 - 学习笔记

## 题目
检测链表中是否存在环（LeetCode 141）

## 原始错误代码

```cpp
bool hasCycle(ListNode* head){
    ListNode* dummy = new ListNode(0);
    dummy->next = head;

    ListNode* slow = dummy;
    ListNode* fast = head;

    while(fast != nullptr && fast->next != nullptr){
        fast = fast->next;        // ❌ 错误：只移动了1步
        if(fast == slow) return true;
        slow = slow->next;
    }

    return false;
}
```

## 错误分析

### 1. 快慢指针移动逻辑错误
- **错误**：fast 每次只移动 1 步（`fast = fast->next`）
- **正确**：fast 应该每次移动 2 步（`fast = fast->next->next`）
- **原因**：Floyd 判圈算法的核心是快指针速度是慢指针的 2 倍，这样才能保证在有环的情况下快指针一定能追上慢指针

### 2. 语法错误
- 第25行缺少分号：`slow = slow->next` 应该是 `slow = slow->next;`

### 3. 不必要的 dummy 节点
- 使用 dummy 节点增加了复杂度，且可能导致逻辑混乱
- 这道题不需要 dummy 节点，快慢指针都从 head 开始即可

### 4. 内存泄漏
- 使用 `new` 创建了 dummy 节点，但没有对应的 `delete`，会造成内存泄漏

## 正确解法

```cpp
bool hasCycle(ListNode* head){
    if(head == nullptr || head->next == nullptr) return false;
    
    ListNode* slow = head;
    ListNode* fast = head;
    
    while(fast != nullptr && fast->next != nullptr){
        slow = slow->next;        // 慢指针移动1步
        fast = fast->next->next;  // 快指针移动2步
        
        if(fast == slow) return true;  // 相遇说明有环
    }
    
    return false;  // fast到达nullptr，说明无环
}
```

## 核心知识点

### 1. Floyd 判圈算法（龟兔赛跑算法）

**原理**：
- 使用两个指针，一个慢指针（每次移动1步），一个快指针（每次移动2步）
- 如果链表有环，快指针最终会追上慢指针（因为快指针比慢指针快1步/轮）
- 如果链表无环，快指针会先到达 nullptr

**时间复杂度**：O(n)
**空间复杂度**：O(1)

**为什么快指针速度必须是慢指针的2倍？**
- 如果速度差为1，那么快指针每轮都会接近慢指针1个节点，最终一定会相遇
- 如果速度差太大（如3倍），可能会跳过慢指针，需要更多轮才能相遇
- 2倍速度是最优选择，既保证能相遇，又不会跳过

### 2. 堆上分配 vs 栈上分配

#### 堆上分配（使用 new）
```cpp
ListNode* dummy = new ListNode(0);
```
- **内存位置**：堆（heap）
- **变量类型**：指针，存储堆上对象的地址
- **生命周期**：由程序员控制，需要手动释放
- **释放方式**：必须使用 `delete dummy;`，否则会内存泄漏
- **使用场景**：需要跨函数使用、生命周期不确定的对象

#### 栈上分配（直接声明）
```cpp
ListNode dummy(0);  // 或 ListNode dummy = ListNode(0);
```
- **内存位置**：栈（stack）
- **变量类型**：对象本身，不是指针
- **生命周期**：随作用域结束自动销毁
- **释放方式**：自动释放，无需手动 delete
- **使用场景**：临时变量、局部辅助对象

#### 使用对比
```cpp
// 堆上分配
ListNode* dummy = new ListNode(0);
dummy->next = head;  // 使用 -> 访问成员
// ... 使用
delete dummy;  // 必须手动释放！

// 栈上分配
ListNode dummy(0);
dummy.next = head;   // 使用 . 访问成员
// ... 使用
// 自动释放，无需delete
```

**建议**：对于临时辅助节点，优先使用栈上分配，避免内存泄漏。

### 3. 边界条件处理

```cpp
if(head == nullptr || head->next == nullptr) return false;
```
- 空链表：无环
- 单节点：无环（除非自己指向自己，但通常题目不会这样）

## 心得总结

1. **理解算法本质**：Floyd 判圈算法的关键是速度差，快指针必须比慢指针快，且速度差为1时最稳定
2. **避免过度设计**：不是所有链表题都需要 dummy 节点，要根据实际情况判断
3. **内存管理**：使用 `new` 必须配对的 `delete`，优先考虑栈上分配
4. **语法细节**：注意分号、指针访问符号（`->` vs `.`）的使用
5. **测试思维**：不仅要考虑小环，还要考虑大环、无环等各种情况
